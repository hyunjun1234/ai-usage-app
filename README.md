# AI Usage

Claude Code · Codex CLI 사용량을 맥북 상단 메뉴 막대에 간소하게 표시하는 앱.

CLI에서는 토큰 사용량·한도를 한눈에 보기 어렵습니다. AI Usage는 두 도구가
로컬에 남기는 기록을 직접 읽어, 메뉴 막대에 사용률을 보여주고 클릭하면
기간별 상세 내역을 펼쳐 보여줍니다.

## 동작 방식

로컬 파일만 읽습니다. 네트워크 통신·로그인 없음.

| 도구 | 데이터 출처 |
|------|-------------|
| Claude | `~/.claude/projects/**/*.jsonl` 의 `message.usage` |
| Codex  | `~/.codex/sessions/**/rollout-*.jsonl` 의 `token_count` 이벤트 |

- 60초마다, 그리고 팝오버를 열 때마다 자동 갱신
- 읽은 바이트 위치를 기억해 새로 추가된 줄만 파싱

## 빌드 & 실행

```sh
cd "Usage App"
./build-app.sh --install        # 빌드 후 /Applications 에 설치
open "/Applications/AI Usage.app"
```

전체 Xcode 없이 Swift 툴체인(Command Line Tools)만으로 빌드됩니다.
`--install` 없이 실행하면 프로젝트 폴더에 `AI Usage.app` 만 생성됩니다.

## 메뉴 막대 조작

- **좌클릭** — 상세 팝오버 열기
- **우클릭** — 빠른 메뉴 (사용량 보기 / 새로고침 / 표시 방식 / 로그인 시 실행 / 종료)

## 사용률(%) — 실시간 vs 추정

| 도구 | 표시 방식 | 이유 |
|------|-----------|------|
| **Codex** | **실시간 한도** (5시간·주간 + 리셋 시각) | `codex` **CLI**가 세션 파일에 실제 한도를 기록함 |
| **Claude** | **예산 추정** | `/usage` 수치는 서버에서 실시간 조회되며 로컬에 저장 안 됨 |

알려진 한계:

- **Codex 데스크톱(Electron) 앱**의 사용량은 디스크에 저장되지 않아 반영되지
  않습니다. `codex` CLI를 쓰면 실시간 한도가 `실시간` 배지로 표시됩니다.
- **Claude**는 실제 한도를 가져올 로컬 경로가 없어, **예산 한도** 칸에
  금액·기준(일간/주간/월간)을 입력하면 그 대비 사용률을 추정합니다.
- 80% 이상 주황색, 100% 이상 빨간색으로 강조됩니다.

## 메뉴 막대 표시 방식

팝오버 하단(또는 우클릭 메뉴)에서 선택:

- **사용률 % (C/X)** — 도구별 사용률 (`C 33%  X 98%`) — *기본값*
- **사용률 % (최대)** — 한도에 가장 가까운 쪽 (`X 98%`)
- **비용 합계 / 비용 (C/X)** — 추정 비용 기준
- **토큰 합계 / 토큰 (C/X)** — 토큰 수 기준

## 종료

우클릭 메뉴 → 종료, 또는 팝오버 우측 하단 종료 버튼.
